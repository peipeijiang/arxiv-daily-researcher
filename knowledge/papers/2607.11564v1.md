<summary><strong>中文摘要</strong></summary>

研究人员将其收集的论文在文献管理工具中组织成个人文件夹层级结构，并将每篇新论文归入其所属文件夹。该任务不同于标准的分层文本分类。用户的文件夹层级并非固定共享的分类体系，而是一种私有的、不断演化的分众分类法（folksonomy），其文件夹含义可能涉及主题、简写、来源或流程导向，且通常由已存储其中的论文所定义。我们将这一场景形式化为个性化分层论文路由（personalized hierarchical paper routing, PHPR）：在无需针对每个用户进行训练的情况下，将新论文分配到用户特定层级中的文件夹。我们提出PaperRouter-Agent，一种无需训练的大语言模型（LLM）智能体，其路由决策不仅基于文件夹名称，更以文件夹成员为依据。该智能体首先缩小候选层级范围，检索文件夹特定证据，通过检查成员论文来验证匹配度，并整合来自用户过去拒绝记录的相似度门控反馈。一项基于真实个人文献库的形成性研究表明，PaperRouter-Agent将整体Recall@1从0.39提升至0.61，Recall@3从0.57提升至0.83，其中在由来源或年份等元数据定义的组织性文件夹上提升最为显著——单次方法在此类文件夹上的Recall@1从0.09降至0.50。在公开的LaMP-2基准测试中，相较于单次基线方法，相同方法将准确率从44.5%提升至51.5%（宏F1值提升9.0），同时保持实际应用中的低成本。

<summary><strong>英文摘要</strong></summary>

Researchers organize the papers they collect into personal folder hierarchies in reference managers, and route each new paper into the folder where it belongs. This task differs from standard hierarchical text classification. A user's folder hierarchy is not a fixed, shared taxonomy but a private and evolving folksonomy whose folder meanings may be topical, shorthand, venue-based, or process-oriented, and are often defined by the papers already stored inside them. We formalize this setting as personalized hierarchical paper routing (PHPR): assigning an incoming paper to folders in a user-specific hierarchy without per-user training. We propose PaperRouter-Agent, a training-free LLM agent that grounds routing decisions in folder members rather than folder names alone. The agent first narrows the candidate hierarchy, retrieves folder-specific evidence, verifies fit by inspecting member papers, and incorporates similarity-gated feedback from past user rejections. A formative study on real personal libraries shows that PaperRouter-Agent raises overall Recall@1 from 0.39 to 0.61 and Recall@3 from 0.57 to 0.83, with the largest gains on organizational folders defined by metadata such as venue or year, where single-shot methods collapses (Recall@1 0.09 to 0.50). On the public LaMP-2 benchmark, the same approach improves accuracy from 44.5% to 51.5% (+9.0 macro-F1) over a single-shot baseline, while remaining low-cost for practical use.

## 深度解读

> 分析依据：**全文深读**

### 核心结论

本文提出个性化分层论文路由（PHPR）任务，即根据用户个人文件夹层次结构将新论文分配到正确文件夹。与标准分层文本分类不同，用户文件夹是私有的、演化的民间分类法，其含义由内部论文定义。作者提出PaperRouter-Agent，一种无需训练的LLM智能体，通过四阶段流程（规划器、检索器、检查器、反射器）实现路由：规划器缩小候选范围，检索器按类型获取证据，检查器阅读成员论文判断匹配，反射器利用用户反馈改进。在真实个人图书馆研究中，Recall@1从0.39提升至0.61，Recall@3从0.57提升至0.83；在LaMP-2基准上，准确率从44.5%提升至51.5%。

### 主要创新

- 形式化个性化分层论文路由（PHPR）任务，区分于标准分层文本分类
- 提出四阶段LLM智能体框架（规划器、检索器、检查器、反射器），无需训练
- 基于文件夹内容而非名称进行路由决策，处理语义模糊、缩写、元数据等文件夹类型
- 引入反馈记忆机制，利用用户接受/拒绝信号在线改进
- 在真实用户图书馆和公共基准上取得显著提升

### 研究方法

采用四阶段LLM智能体：规划器通过自上而下遍历缩小候选文件夹；检索器按文件夹类型（主题/元数据）选择证据；检查器阅读成员论文判断匹配，支持弃权；反射器存储用户反馈并注入相关拒绝案例。使用gpt-4o-mini作为骨干模型，在真实Zotero图书馆和LaMP-2基准上评估。

### 关键结果

在5位研究人员的真实Zotero图书馆上，Recall@1从0.39提升至0.61，Recall@3从0.57提升至0.83；在LaMP-2基准上，准确率从44.5%提升至51.5%，宏F1提升9.0点。

### 技术栈

- GPT-4o-mini
- Zotero插件（JavaScript和Python）
- OpenRouter API
- 嵌入API（用于向量相似度）
- Python

### 方法优势

- 无需训练，直接使用预训练LLM
- 处理多种文件夹类型（主题、缩写、元数据、状态）
- 通过内容阅读和反馈记忆实现个性化
- 在真实用户场景和公共基准上均有效

### 主要局限

- 反射器效果仅在单次研究中体现，未进行纵向评估
- 缺乏大规模公共基准
- 处理状态文件夹（如“待读”）仍为困难情况
- 成本与延迟高于单次调用方法

### 与当前研究方向的关联

论文直接涉及个性化文本分类、检索增强生成、LLM智能体和个人知识管理，与关键词高度相关。
